# HANDOFF — x-sun clean (для нового чату / іншого агента)

Прочитай цей файл першим. Він дає повний контекст проєкту, щоб не перечитувати весь код.

## Що це
Сайт клінінгової компанії **x-sun clean** (Познань, Польща). Аудиторія — українці в Польщі.
Стек: **статичний HTML + CSS + vanilla JS**, без збірки. Запуск: `python3 -m http.server 4173`.
Мова інтерфейсу — українська (`<html lang="uk">`). Плановані PL/EN — «незабаром».

## Сторінки
| Файл | Що |
|---|---|
| `index.html` | Головна: hero з міні-калькулятором, переваги, послуги (горизонт. скрол з пінінгом GSAP), секція «Що входить у прибирання» (інтерактивні кімнати), ціни, «до/після» (галерея), «Як це працює», зона (карта+CTA), для бізнесу, відгуки (з навігацією), FAQ, футер, плаваюча кнопка |

| `shcho-prybyraiemo.html` | Окрема сторінка «Що входить у прибирання» (та сама секція, що й на головній) |

| `dyzajn-systema.html` | Живий гайд дизайн-системи (усі токени й компоненти) |

## JS (усі у `js/`)
- `config.js` — **КОНТАКТИ Й НАЛАШТУВАННЯ** (`window.XSUN`): phone, telegram, whatsapp, email, promoCodes, leadEndpoint.
- `nav.js` — телефон/email/telegram у розмітці, перемикач мови (спільний).
- `order.js` — уся логіка сторінки замовлення (калькулятор, валідація, повідомлення, заявка).
- `home.js` — головна (hero-калькулятор, ціни, галерея до/після, відгуки, GSAP-анімації).
- `what.js` — інтерактивні кімнати «Що входить» (дані хотспотів усередині, працює і на головній, і на окремій сторінці).
- `fx.js` — магнітні кнопки (клас `.magnet`).
- `cookie.js` — GDPR-банер згоди (блокує скрол + затемнення до вибору).

## CSS
- `css/styles.css` — **дизайн-система (токени в `:root`)** + спільні стилі (кнопки, форми, футер, cookie, сторінка замовлення).
- `css/home.css` — стилі головної.
- Усе прив'язано до токенів: кольори, `--font-display`/`--font-body`, `--radius-*`, `--shadow-*`, `--ease`. Щоб змінити щось глобально — правити `:root`.

## Дизайн-мова
- Кольори: тепла кремова база + один жовтий бренд-акцент `--accent #FFC53D`. Чорнило `#141414` (не чистий чорний).
- Шрифти: **Unbounded** (заголовки) + **Golos Text** (текст).
- Кнопки: `.btn` + `.btn-primary/secondary/ghost/ghost-inv` + розмір `.btn-sm/базовий/.btn-lg` + `.btn-block`, `.magnet`.

## Бізнес-логіка калькулятора (order.js)
- Базова ціна квартири: `184.90 + (кімнати−1)*45 + (санвузли−1)*40`. Будинок `×1.2`. Кухня-студія `−10 zł`. Після ремонту — `6 zł/м²`.
- Періодичність (порядок і дефолт): **Разове (0%)** → місяць (−10%) → два тижні (−15%) → тиждень (−20%).
- **Мінімальна тривалість прибирання — 3 години** (бізнес-правило).
- Миття вікон — **поштучно** (лічильник у картці опції), інші опції — перемикачі.
- Сводка показує «якірну» ціну: перекреслена стара + актуальна + бейдж вигоди (лише коли є реальна знижка).
- Дві дії: **Замовити в Telegram** (основна, повідомлення заповнюється саме) + **Надіслати заявку** (запасна, для тих без Telegram) — обидві у сводці й унизу форми.

## ⚠️ ПЕРЕД ПУБЛІКАЦІЄЮ (обов'язково)
1. **Контакти в `js/config.js`**: `phone` (зараз `+48 XXX XXX XXX`), `telegram`, `whatsapp`, `email` — усе заглушки.
2. **`leadEndpoint`** порожній → кнопка «Надіслати заявку» лише просить зателефонувати. Створити форму на formspree.io і вставити URL, щоб заявки йшли менеджеру.
3. **`privacy.html`** — шаблон, потребує **юриста** й реальних реквізитів (назва, адреса, NIP/REGON — позначені `[вкажіть …]`).
4. **Фото плити «після»** (`img/ba2-after.webp`) має чужий вотермарк «Freshroom» — замінити на своє фото.
5. **Промокод** `START15` (−15%) у config — тестовий, перевірити/замінити.
6. **Відгуки** на головній — демонстраційні тексти, замінити на реальні.
7. **Кеш**: для деплою додати версії до CSS/JS (`styles.css?v=1`) — браузери агресивно кешують.

## Робочі нюанси
- **Панель предпросмотра флакі**: програмний скрол/навігація часто «не липнуть» або перескакують на іншу сторінку. Це не баг коду — перевіряти computed-стилі програмно, а не лише скріншотами.
- Емодзі-прапори в селекті коду країни на Windows/Chrome показуються буквами (обмеження ОС) — на mac/iOS/Android видно прапори.

## Дизайн-вподобання замовника (важливо для стилю)
- Любить делікатні, ледь помітні ефекти; не любить нав'язливі/різкі (магніт зробили слабким ~0.07, ховери — лише тонкий натяк рамкою, без заливки).
- Сам вирізає фон зображень і надсилає готові — підключати як є.
- Цінує маркетингові прийоми (якірна ціна, вигода), чистий UX без перевантаження.
